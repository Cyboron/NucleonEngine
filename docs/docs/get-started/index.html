Get Started

Let's setup and use Nucleon Engine in less than 5 minutes.

Getting Started#

Get started by downloading, cloning or forking Nucleon Engine on GitHub.
In your project, create an Empty GameObject and assign the Nucleon Manager component to it.

This step is essential! We are ready to start adding actual physics now.

Adding Colliders#

Create a Cube under 3D Object in the hierachy and set its position to 0, 0, 0 (x, y, z).


You should now see a normal cube in your scene.
Next up, remove Unitys Box Collider component and add a Nucleon Box Collider component to the cube.


Once you have added the Nucleon Box Collider, you will see a green box overlapping the cube:

Congratulations, you have just setup your first Nucleon Collider!

Let's repeat the step. Scale one of the two cubes with the vector 8, 0.1, 8 (x, y, z).
Change the other cubes position to 0, 4, 0 (x, y, z) but let its scale remain.


Your scene should now contain a cube (/quader) respresenting the ground,
and a cube that'll represent the player, both containing a Nucleon Box Collider.
Lets continue adding actual physics in the next section!

Adding Physics#

Physics normally represent any kind of dynamic movement in games.
When thinking of physics poeple usually remind themselves to gravity or forces-
So lets go ahead and add some!

We'll start by adding the Nucleon Rigidbody component to the cube representing
the player we setup above. We'll use it's default configuration for now.


Now lets run the game and see the magic happening! Actually its not magic, its physics.

Did we just add gravity to the cube? Yes we did! Gravity is cool, actual movement is cooler,
so lets add some actual movement to the cube. Create a new script, assign it to the Cube
and paste the following code inside the class:

public float Speed = 5;
private NucleonRigidbody Body;
void Awake()
{
Body = GetComponent<NucleonRigidbody>();
}
void Update()
{
Body.Velocity = new svector3((sfloat)Input.GetAxis("Horizontal") * (sfloat)Speed, (sfloat)0, (sfloat)Input.GetAxis("Vertical") * (sfloat)Speed);
}

Does that look complex? No worries, it isn't, we'll break it down soon.
Run the game and move around with WASD. Make sure you didn't forget to assign the script to the cube!

How does that actually work? Its quite simple. In the Awake() function we are referencing our
NucleonRigidbody Component to the Body variable of the script. In the Update() function
we are setting the Velocity of our cube/player. We use a svector3* for that, taking 3 sfloat's* as parameters.

*sfloat and svector3 explained

When calculating stuff with floats, any hardware will calculate a slightly different result due to CPU differences.
A sfloat (32 bit floating point) will always calculate the same result, same goes for a svector3.

Make sure to convert your floats into sfloats before calculating physics!

The x axis of our velocity will be set to the Horizontal input of our player (left/right --> A/D),
the y axis will be set to 0 as we are not going to change our up and down movement.
Finally we'll set the z axis to the Vertical input (forward/backward --> W/S) and there we are!

You just setup Colliders and Physics using Nucleon Engine within a few minutes, Good Job!
